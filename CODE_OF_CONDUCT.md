# Code of Conduct

## Our Commitment

Psychometricist-AI is an open scientific project committed to fostering
a respectful, rigorous, and collaborative research-oriented environment.

We are committed to ensuring a harassment-free experience for everyone,
regardless of age, body size, disability, ethnicity, gender identity,
level of experience, nationality, personal appearance, race, religion,
or sexual identity.

Scientific disagreement is welcome.
Personal attacks are not.

---

## Expected Behavior

Participants are expected to:

- Use respectful and professional language
- Provide evidence-based arguments when discussing methodologies
- Clearly distinguish opinion from validated findings
- Accept constructive criticism
- Focus discussions on improving scientific rigor and reproducibility

---

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment or discriminatory language
- Personal attacks
- Dismissive or non-constructive criticism
- Misrepresentation of scientific claims
- Sharing confidential or private information without permission

---

## Scientific Integrity Clause

Given the scientific nature of this project:

- Claims must be supported by references or empirical validation
- Contributors must avoid overstating findings
- Algorithmic or psychometric modifications must include justification

Maintainers may reject contributions that compromise methodological integrity.

---

## Enforcement Responsibilities

Project administrators are responsible for:

- Clarifying standards
- Moderating discussions
- Enforcing this Code of Conduct fairly and consistently

---

## Reporting Issues

If you experience or witness unacceptable behavior, please contact the maintainers
via GitHub Issues marked as confidential or via direct communication.

All reports will be handled with discretion.

---

## Enforcement

Maintainers may take appropriate corrective action, including:

- Warning
- Temporary restriction
- Permanent removal from project spaces

---

This Code of Conduct applies to:
- GitHub Issues
- Pull Requests
- GitHub Discussions
- Project communication channels
